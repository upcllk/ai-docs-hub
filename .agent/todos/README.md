## 功能缺陷

### [TODO-001] 多行选中标注不显示高亮也不支持跳转

- **状态**：🚫 暂不处理（后续迭代）
- **发现时间**：2026-06-28
- **现象**：用户选中跨越多个段落/列表项的文字后提交评论，刷新页面时该标注在文档中**不会高亮显示**，也不支持点击跳转
- **根本原因**：详见 `.agent/pitfalls/README.md` [PITFALL-007]。
  `TreeWalker` 每次只遍历单个文本节点，多行选中的 `selectedText` 跨越多个节点，无法在单节点内整体匹配
- **临时措施**：当前代码仅高亮 `selectedText` 的第一行，多行评论在侧边面板正常显示，但文档中无对应高亮
- **建议解决方案**：
  - 方案 A：选中时记录 DOM 路径 + 节点内偏移量（精确，但文档结构变动后失效）
  - 方案 B：选中时记录在 `docContainer.innerText` 中的字符偏移量，渲染时用 `Range` API 重建（对纯文本变动鲁棒）
  - 方案 C：限制用户只能选单行文字（用户体验较差）
- **影响范围**：仅影响多行选中的标注高亮，单行标注功能正常
